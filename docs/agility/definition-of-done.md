# Definition of Done

The Definition of Done defines the minimum criteria that a User Story must satisfy before it can be considered complete.

This is a living document and is not intended to be exhaustive.
It will be updated as the project evolves.

## General
- [ ] The feature is available through the user interface unless explicitly stated otherwise.
- [ ] All the requirements described in the "how to validate" section are met (all the boxes are checked)
- [ ] The applicable transversal requirements are met (see [Transversal-requirements.md](../specs/Transversal-requirements.md))

## Tests
- [ ] Appropriate automated tests have been added or updated.
- [ ] All automated tests pass in the CI pipeline.
- [ ] Any deactivated test is documented with a comment explaining the reason.

## Code review
- [ ] The code has been reviewed by at least one other developer when available.
- [ ] If no reviewer is available, the PR is not blocked for this reason.
